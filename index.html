Practice Git Commands

1. Creating working directory on computer

We create the project directory. Inside that directory right click and select Git Bash. To initialize our project we type in git bash console git init. After that project directory will be initialized for GitHub. Now we need to add our credentials , user name and user email. To do that we type in git bash console git config --global user.name "Your Name" and git config --global user.email "your@email.com". This will associate our local files with GitHub repository.

2. Create some test files in project directory

Right click in directory and create some file, for example index.html. But we can also create file from git bash console, to do that we type touch name-of-the-file.extension. This command should create new file.

3. Checking status

To check status simply type in git bash git status. This will show current status of files.

4. Before we continue...

Now it is important to understand that all files are tracked in stages. First we have a working directory with files. Then we need to add them, prepare them. To prepare files we type git add . . This will prepare all files in directory. If we want seperate files to be added we type git add myFile.extension. Now in the first command we used dot, to tell that we want all files to prepare. Just to be sure in what is going on we will type git status. This is good practice to be aware of process. Next thing is to commit those files. To do that we type git commit -m "comment , message , short description of changes we made.". Again we check our status git status.

5. GitHub Repository

Create your GitHub repository under your GitHub account. This will be place where we put our files. Now we need to clone our project, and to do that we type git clone URL-LOCATION-OF-REPOSITORY.
